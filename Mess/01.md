## 一、AI Agent 岗位全景图

### 1.1 岗位分类

| 类型 | 典型岗位名 | 核心要求 |

| **Agent 应用开发** | AI Agent 开发工程师 / 大模型应用开发 | 工程能力 > 算法能力；LangChain/LangGraph 熟练 |

### Phase 1：Agent 核心基础（第 3-6 周）

> 目标：理解 LLM Agent 核心机制，能调用 API 构建最简单的 Agent

#### 📚 学习内容

**1. LLM API 调用（第 1 天）**
```python
# 你的第一个 API 调用
import openai  # 或 anthropic

client = openai.OpenAI(api_key="sk-xxx")
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello"}]
)
print(response.choices[0].message.content)
```

**2. 理解 Agent 核心模式（第 2-5 天）**
- **ReAct 模式**：Reasoning + Acting → Observation 循环
- **Tool Calling**：LLM 调用外部工具
- **Agent 循环**：Think → Act → Observe → Think...

**3. Prompt Engineering 基础（第 6-8 天）**
- System Prompt 设计
- Few-shot 示例
- Chain-of-Thought
- 结构化输出（JSON mode / Tool use）

**4. OpenAI Function Calling / Anthropic Tool Use（第 9-12 天）**
```
概念：工具定义 → LLM 选择工具 → 解析参数 → 执行 → 返回结果
```

**5. LangChain 入门（第 13-20 天）**
- Model I/O：Prompt Template, Output Parser
- Memory：ConversationBufferMemory
- Chains：LLMChain, SequentialChain
- Agents：Tool, AgentExecutor

#### 💻 核心项目

**项目 1-A：命令行助手（第 1 周）**
```
功能：通过自然语言执行系统命令、文件操作
技术：OpenAI API + Tool Calling + Python
技能点：API 调用、工具定义、函数调用解析

学到：
  ✅ LLM API 调用流程
  ✅ Function Calling 全流程
  ✅ 工具定义与参数解析
```

**项目 1-B：个人知识问答机器人（第 2 周）**

技能点：LangChain 基础 + 文档加载

学到：
  ✅ LangChain 框架流程

```

**项目 1-C：自动周报生成器（第 3 周）**

### Phase 2：RAG + 工具链实战（第 7-10 周）

> 目标：构建一个生产级别的 RAG 系统，理解检索增强生成的全流程

#### 📚 学习内容

**1. 向量数据库（第 1-5 天）**
```
- Embedding 模型选择（text-embedding-3-small, bge, etc.）
- 向量存储与检索（ChromaDB / FAISS / Milvus）
- 相似度计算（cosine / dot product / euclidean）
- 元数据过滤
```

**2. 文档解析（第 6-10 天）**
```
- PDF 解析（PyMuPDF, pdfplumber）
- 文档分块策略（chunk_size, overlap, semantic chunking）
- 多模态文档处理（图片提取 OCR）
```

**3. 检索增强（第 11-15 天）**
```
- Embedding + Keyword 混合检索（Hybrid Search）
- 重排序（Re-ranking）
- Query 转换（Query Rewrite / HyDE）
- RAPTOR：递归摘要检索
```

**4. LangChain + RAG 深度使用（第 16-20 天）**
```
- RetrievalQA Chain
- 多轮对话 + 检索 + 记忆
- Contextual Compression
- Self-Query Retriever
```

#### 💻 核心项目

**项目 2：智能知识库系统**
```
功能：
  - 📄 上传 PDF/Markdown/网页 → 自动解析
  - 🔍 自然语言搜索文档内容
  - 💬 基于文档的多轮对话
  - 📊 展示检索到的相关片段 + 来源

技术栈：
  Python + LangChain + ChromaDB + FastAPI + Streamlit

目录结构建议：
  smart-knowledge-base/
  ├── backend/
  │   ├── loader/          # 文档加载器
  │   ├── splitter/        # 文档分块
  │   ├── embeddings/      # 向量化
  │   ├── retriever/       # 检索逻辑
  │   ├── reranker/        # 重排序
  │   └── api/             # FastAPI 端点
  ├── frontend/            # Streamlit UI
  └── tests/               # 测试

学到：
  ✅ 完整 RAG 流水线
  ✅ 不同分块策略的效果对比
  ✅ 检索质量评估（Recall / Precision）
  ✅ FastAPI 服务部署
  ✅ 简单的 UI 交互
```

#### ✅ Checkpoint
- [ ] 理解 Chunking 策略对检索效果的影响
- [ ] 能用两种以上检索方式做 Hybrid Search
- [ ] 能评估 RAG 系统质量（有过测试数据）
- [ ] 项目代码完整、文档齐全

## 七、关键提醒

### ⚠️ 避坑指南

1. **不要只调 API**：面试官要的是你对原理的理解，不是调包侠
2. **不要抄项目**：把别人的项目改一改就说是自己的 → 面试一问就露馅
3. **不要只写代码不写文档**：README 就是你的脸面
4. **不要等到"准备好"再投**：永远没有完全准备好的时候，先投、边面边学
5. **不要忽视八股**：虽然是 AI Agent 岗，计算机网络、操作系统、数据库基础依然会问
6. **不要忽视算法**：大厂第一轮基本都是算法，挂了就什么都没有了

### ✅ 成功的项目 README 模板

```markdown
# 项目名称

## 简介
一两句话说清楚这个项目是做什么的

## Demo
（GIF 截图 / 录屏链接）

## 技术栈
Python 3.11, LangGraph, ChromaDB, FastAPI, Docker

## 效果数据
- 检索准确率：92%
- 平均响应时间：1.2s
- Token 消耗减少：35%（对比 baseline）

## 快速开始
```bash
git clone ...
cd ...
pip install -r requirements.txt
cp .env.example .env
python main.py
```

## 架构图
（用 Mermaid / Draw.io 画）

## 项目结构
（tree 命令输出）

## 学到了什么
- 3-5 个关键技术点
- 遇到的坑与解决方案
```

### 🎯 你只需要做到这几点就能脱颖而出

1. **完整的 GitHub 项目**（README 完善 + 代码整洁）
2. **能流畅讲清楚每个项目的技术细节和难点**
3. **LeetCode 200 题以上的算法储备**
4. **对 Agent 的核心模式有深入理解**（不是只会调框架）
5. **持续学习的态度**（关注最新论文、框架更新）
